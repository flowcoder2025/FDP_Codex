## Recommendation

Recommended choice: A, user-chosen mnemonic code.

Use this when the target project is intended to be serious, long-lived, public, or contributor-facing.

Proposed accepted rule if A is chosen:

```text

```

Required fields for the target manifest:

- `project_scope_code`
- `scope_code_status: accepted`
- `scope_code_decision_ref`

Code constraints:

- 2-6 uppercase alphanumeric characters.
- Starts with a letter.
- Must not be `CX`.
- Unique among target projects managed by the same FDP_Codex workspace.

## Fallback

Fallback choice: B, temporary `TG` code.

Use this only when speed matters more than long-term traceability or the target project is unnamed.

Required debt if B is chosen:

- Record `scope_code_status: temporary` in the target manifest.
- Add migration debt before public release, external contribution mode, or long-lived automation.
- Rename outstanding target WIs before crossing those repayment triggers.

## Other Options

C, repository-derived code, is acceptable when the repository name is stable and distinctive. The derived code still needs explicit confirmation before scaffold generation.

D, long explicit slug, is best for portfolios with many similar target projects, but it is heavier in branches, PR titles, and handoffs.
